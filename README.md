# Armies of Terra Map

Этот проект собирает общую карту сервера из тайлов JourneyMap и отображает города на веб-карте.

## Что есть в проекте

- `scripts/merge_journeymap_tiles.py` объединяет тайлы карты и пересобирает `tiles.json` и preview-картинки.
- `scripts/import_city_waypoints.py` импортирует городские метки `town*` из JourneyMap в `data/cities.json`.
- `index.html` показывает карту, таймлайн и фильтры городов.

## Источник меток городов

Скрипт городов читает waypoint-файлы из папки:

```text
\curseforge\minecraft\Instances\Armies of Terra\journeymap\data\mp\Armies~of~Terra\waypoints
```

Берутся все файлы `town*.json`.

## Правила именования меток городов

Рекомендуемый формат имени метки в игре:

```text
town <CityName> nation <NationName>
```

Примеры:

```text
town NovyGrad nation Warbound
town Corvinburg nation Avernia capital
town FortCreed bankrupt
town Unknown ruin
```

Поддерживаемые маркеры:

- `nation <NationName>`: назначает нацию городу
- `capital`: помечает город как столицу
- `ruin`: помечает город как руины
- `bankrupt`: тоже трактуется как руины
- `shop`: сохраняется как тег

Важно:

- `capital` не входит в имя нации
- столица и обычные города одной нации получают один и тот же цвет
- столицы на карте рисуются звездой

## Игровое время

История городов привязана к внутриигровому году.

Формат года:

```text
ровно 4 цифры
```

Пример:

```text
1408
```

Год `1408` считается стартовой точкой сбора данных.

Если `--game-date` не указан, скрипт использует `1408`.

Это сделано специально, чтобы на этапе первичного заполнения все стартовые данные попадали в один и тот же год и не дробились случайно по разным датам.

## Импорт городов

### 1. Обычный импорт текущего состояния

Команда:

```powershell
python scripts/import_city_waypoints.py "C:\Users\atfas\curseforge\minecraft\Instances\Armies of Terra\journeymap\data\mp\Armies~of~Terra\waypoints" data\cities.json
```

Поведение:

- читает актуальные `town*.json`
- полностью пересобирает `data/cities.json`
- записывает все города как состояние на год `1408`
- удаляет из `data/cities.json` города, которых больше нет среди текущих waypoint-файлов

Этот режим нужен для первичного накопления и чистки актуального набора.

### 2. Обычный импорт с явным игровым годом

Команда:

```powershell
python scripts/import_city_waypoints.py "C:\Users\atfas\curseforge\minecraft\Instances\Armies of Terra\journeymap\data\mp\Armies~of~Terra\waypoints" data\cities.json --game-date 1408
```

Поведение то же самое, но год задаётся явно.

### 3. Фиксация снимка в историю

Команда:

```powershell
python scripts/import_city_waypoints.py "C:\Users\atfas\curseforge\minecraft\Instances\Armies of Terra\journeymap\data\mp\Armies~of~Terra\waypoints" data\cities.json --history --game-date 1409
```

Поведение:

- читает текущее состояние waypoint-файлов
- сравнивает его с последним состоянием городов в `data/cities.json`
- если у города изменились нация, статус, имя, столица или теги, закрывает старое состояние годом `1409` и открывает новое
- если город исчез из waypoint-файлов, закрывает его последнее состояние годом `1409`
- если появился новый город, добавляет его с началом в `1409`
- обновляет `timeline.dates`

Важно:

- `--history` не удаляет старые состояния
- `--history` именно накапливает историю

## Когда какой режим использовать

Пока идёт первичный сбор:

- поддерживай в игре актуальные метки
- запускай обычный импорт
- при необходимости много раз перезаписывай `data/cities.json`
- все стартовые данные будут сидеть в `1408`

Когда начнётся фиксация изменений по ходу истории мира:

- сначала обновляй метки в игре до актуального состояния на нужный год
- затем запускай импорт с `--history --game-date <год>`

## Что считается изменением состояния города

Исторический снимок создаёт новое состояние, если изменилось хотя бы одно из полей:

- `status`
- `nation_id`
- `label`
- `tags`
- `is_capital`

Также состояние считается изменённым, если поменялись координаты города.

## Что происходит, если метка исчезла

Сейчас поведение такое:

- в обычном импорте город просто пропадает из итогового `data/cities.json`
- в режиме `--history` последнее состояние города закрывается указанным игровым годом

Это специально сделано для безопасности:

- пока идёт первичная чистка и переименование меток, удобнее иметь полностью актуальный файл
- позже, в историческом режиме, исчезновение уже не удаляет прошлое, а фиксируется как изменение

## Фильтры на карте

На карте доступны фильтры:

- `World border`
- `Show capitals`
- `Show national cities`
- `Show free cities`
- `Show ruins`
- отдельные чекбоксы по каждой нации

Фильтр нации скрывает все города этой нации, включая её столицу.

## Граница мира

На карте отображается квадратная граница мира. По умолчанию она проходит по координатам `-10000` и `10000` по обеим осям, то есть рисуется квадрат:

- от `X = -10000` до `X = 10000`
- от `Y = -10000` до `Y = 10000`

Настройка вынесена в объект `WORLD_BORDER_CONFIG` в `index.html`.

Если границу мира расширят, достаточно поменять эти параметры:

- `minX`
- `maxX`
- `minY`
- `maxY`

Дополнительно там же можно быстро настроить внешний вид линии:

- `color`
- `weight`
- `opacity`
- `dashArray`

## Команды для тайлов карты

По умолчанию `scripts/merge_journeymap_tiles.py` берет данные из двух источников:

- твои локальные тайлы JourneyMap из папки игры
- входящие файлы из `imports\journeymap_inbox`

В `imports\journeymap_inbox` можно класть:

- `.zip` архивы с тайлами
- папки с тайлами
- отдельные файлы `x,y.png` прямо в корень inbox

После успешного merge скрипт автоматически удаляет обработанные архивы, папки и отдельные `png` из inbox.

Основной merge:

```powershell
python scripts/merge_journeymap_tiles.py
```

Проверка без записи:

```powershell
python scripts/merge_journeymap_tiles.py --dry-run
```

Не удалять обработанные архивы и папки из inbox:

```powershell
python scripts/merge_journeymap_tiles.py --keep-inbox
```

Этот флаг также сохраняет отдельные `png`-тайлы, если они лежат в inbox без архива.

Не пересобирать preview:

```powershell
python scripts/merge_journeymap_tiles.py --skip-previews
```

## Локальный запуск карты с прямым сохранением geography.json и cities.json

Для обычного просмотра можно использовать любой статический HTTP server.

Если нужен безопасный `Save Geo` и `Save Cities` без диалога выбора файла, запускай локальный сервер проекта:

```powershell
python scripts/local_map_server.py
```

После этого открой:

```text
http://127.0.0.1:8000
```

В этом режиме:

```text
data/geography.json
data/cities.json
```

- кнопка `Save Geo` сохраняет только в `data/geography.json`
- кнопка `Save Cities` сохраняет только в `data/cities.json`
- редакторы geography и cities доступны только локально

### Что умеет локальный City Editor

- выбирать город из списка или кликом по метке на карте
- создавать новый город на текущем году таймлайна
- менять имя города, координаты и основателя
- менять для активного года статус, столицу, нацию, теги и мэра
- создавать новое состояние города на текущем году через `Split/Add state`
- удалять состояние или весь город
- открывать и использовать встроенный `Nation Editor`
- сохранять итоговый `data/cities.json` прямо из браузера

Важно:

- `Founder` хранится один раз на уровне города
- `Mayor` хранится внутри конкретного состояния и привязан к таймлайну
- если хочешь поменять статус или мэра только с определённого года, сначала переключи таймлайн на этот год и используй `Split/Add state`

### Что умеет локальный Nation Editor

- создавать новую нацию прямо из браузера
- менять название нации, цвет и год основания `founded`
- переименовывать `id` нации через смену имени без ручной правки JSON
- автоматически переносить все связанные `city.states[].nation_id` на новый `id`
- удалять нацию с отвязкой связанных городов

Важно:

- год основания нации использует тот же игровой формат: ровно `4` цифры
- на выбранном году нация считается существующей, только если у неё есть хотя бы один город
- фильтры на карте показывают только нации, у которых есть города на активном году таймлайна

## Полезная практика

- Для первичного периода всегда держи год `1408`.
- Перед исторической фиксацией сначала убедись, что метки в игре отражают ровно нужный внутриигровой год.
- Если сомневаешься, сначала сделай копию `data/cities.json`, потом запускай `--history`.
- Не используй для года ничего кроме 4 цифр, иначе скрипт завершится с ошибкой.
